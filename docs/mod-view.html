Handle a Ticket

Follow these steps to handle tickets properly:

  • Initial Check: Ensure required information: Minecraft name, client, mods, problem.
  • Responding: Politely decline item/money requests; redirect suggestions to server-suggestions.
  • Issues: Identify the problem type, check plugins in-game with /plugins, and involve admins if necessary.
  • Closing: Record details like problem, solution, and a mod note.

Handle Bug Reports

Steps for managing bug reports:

  • Initial Check: Collect information: Minecraft name, bug description, client, mod, version, rank, etc.
  • Responding: Decline item/money requests; redirect questions to q-a.
  • Resolving: Attempt fixes, consult Google if needed, or ping @triggerhappy_.
  • Archiving: Mark bugs resolved with the /buglist command.

Apply for Ranks

Process for handling rank applications:

  • Applicant Info: Ensure all required details: role applied for, Minecraft name, Discord name, age, playtime.
  • Eligibility: Age 15+ for roles above Mod, 1-week+ playtime.
  • Decision: Review applicant's story and reasons for applying.
